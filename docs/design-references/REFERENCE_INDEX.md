# Design Reference Index

This directory stores raw external design references for RiskShield AI.
They are reference material only, not brand copies.

## 1. xAI

- Source URL: https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/x.ai/DESIGN.md
- Good fit for RiskShield AI:
  - Near-black canvas and monochrome restraint
  - Pill-shaped controls and thin borders
  - Sparse, high-contrast headline hierarchy
  - Technical, futuristic, security-adjacent tone
- Poor fit for RiskShield AI:
  - Anything that makes the product feel empty or under-informative
  - Pure marketing minimalism without enough dashboard density
  - Playful stickers, mascots, or casual SaaS decoration
- Weight in RiskShield AI:
  - Primary visual baseline for marketing, auth, and shell chrome

## 2. Sentry

- Source URL: https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/sentry/DESIGN.md
- Good fit for RiskShield AI:
  - Dense, security-forward dashboard surfaces
  - Clear data hierarchy and alert states
  - Strong module containers for scan results and histories
- Poor fit for RiskShield AI:
  - Overly playful branding
  - Loud illustrations that reduce trust
  - Unnecessary visual noise in operational tables
- Weight in RiskShield AI:
  - Secondary reference for dashboard, scan results, and operational surfaces

## 3. Vercel

- Source URL: https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/vercel/DESIGN.md
- Good fit for RiskShield AI:
  - Developer-product clarity
  - Clean API/access presentation
  - Precise code and docs treatment
  - High-end platform polish
- Poor fit for RiskShield AI:
  - Overly airy consumer marketing layouts
  - Generic SaaS gradients that do not support a security posture
- Weight in RiskShield AI:
  - Secondary reference for API pages, docs, and developer tooling

## 4. Stripe

- Source URL: https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/stripe/DESIGN.md
- Good fit for RiskShield AI:
  - Clear pricing and billing hierarchy
  - Strong conversion emphasis around plans and usage
  - Compact, premium, finance-adjacent layout discipline
- Poor fit for RiskShield AI:
  - Bright consumer marketing treatment that dilutes the security feel
  - Pricing pages that hide important constraints or billing details
- Weight in RiskShield AI:
  - Secondary reference for pricing, billing, quota, and upgrade flows
